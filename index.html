Introduction

The work we attempted to reproduce in this project is the work done by researchers at Columbia University in 2023, a text-to-speech model called StyleTTS 2. StyleTTS 2 itself is an improvement on StyleTTS, a model created in 2022 by Columbia University. The source code for StyleTTS 2 can be found here.

StyleTTS made substantial contributions to TTS research, particularly in achieving more natural and expressive speech synthesis. By explicitly modelling speech style, StyleTTS allows for the creation of more personalized and expressive TTS applications. Its efficiency and modular architecture also set a foundation for subsequent advancements in the field, including those realized in StyleTTS 2.

In this work, we build upon a released StyleTTS 2 model by fine-tuning it on a French dataset, to attempt to produce a French TTS model of a similar quality as published in the original StyleTTS 2 paper.

Background

Original StyleTTS features

The original model has the following main features: Style-based generation, a non-autoregressive framework, a modular design and a specific training and optimisation method. These are explained in more detail in the following sections.

Style-Based Generation

As mentioned previously, StyleTTS models speech style directly by using a style encoder which captures elements such as intonation, rhythm, and emotional tone. This is then made into a “style vector”, which is extracted by the encoder and used when generating speech, giving the user a much more human-like experience.

Non-Autoregressive Framework

Another innovation from StyleTTS includes its non-autoregressive architecture, meaning that it generates the entire mel-spectrogram (a representation of the spectrum of frequencies in a sound (as interpreted by humans) at the same time, instead of one time-step at a time. This led to faster inference times when compared to other TTS models, speeding up the synthesis process without decreasing the quality of the generated audio.

Modular Design

Also mentioned previously, StyleTTS has a modular design split into 3 main components: speech generation system, TTS prediction system and utility system for training. This division of tasks allows users to realize which part of the model may need more training. For instance, if the pitch or duration of the words seems to be off, one would know to take a better look at the TTS prediction system as it is in charge of predicting prosodic elements such as those.

Training and Optimization

Finally, StyleTTS was trained in 2 stages. Firstly, the acoustic models responsible for the mel-spectrogram are trained. This phase focuses on accurately mapping from phonemes and style vectors to proper mel-spectrograms. In the second stage, the model’s prediction abilities are addressed, particularly the duration and prosody prediction. This training scheme allows the model to effectively capture both the linguistic content and the style elements of the speech.

StyleTTS 2

 

With StyleTTS 2, the research team at Columbia University built upon this foundation, introducing innovations such as end-to-end training, direct waveform synthesis, and adversarial training with large speech language models. These improvements not only enhance the naturalness and expressiveness of synthesized speech but also highlight the ongoing evolution of TTS technology towards more human-like and versatile systems. There are several differences between StyleTTS and StyleTTS 2 which are explained below.

End-to-End Training

In StyleTTS 2 an end-to-end training process is introduced. The typical TTS architecture is modified by integrating decoders that directly generate waveforms from phonetic inputs and style inputs. This allows for the usual mel-spectogram projections step to be skipped.

Direct Waveform Synthesis and Adversarial Training

StyleTTS 2 incorporates direct waveform synthesis, eliminating the need for a separate vocoder step to convert mel-spectrograms into audio waveforms. Additionally, it utilises adversarial training methods, with large speech-language models acting as discriminators to enhance the natural and realistic sound of the synthesised speech.

Performance Enhancements

Through its advanced methodologies, StyleTTS 2 achieves significant improvements in speech quality, naturalness, and expressiveness. It sets new benchmarks in human-level speech synthesis on various datasets and showcases superior performance in zero-shot speaker adaptation tasks.

Objective & Implementation

The StyleTTS 2 model works perfectly with English and is a great advancement in that respect. However, a text-to-speech model would excel if it was capable of multilingual text-to-speech, but this is a rather significant shortcoming of StyleTTS 2, and still a problem under review in ongoing research[1].

Dataset

Our objective thus consisted of training the model in a language other than English, as that was the language that it was trained and tested on. This starts with finding a dataset in another language with text files and sound files. 

PL-BERT

It was found online that simply replacing the dataset would not be enough, additionally a new PL-BERT model would have to be added[2]. Fortunately, a multilingual model to be used for StyleTTS2 has been developed by the community for 14 languages and can be found here. Once PL-BERT is replaced with a multilingual model, the new dataset can be inserted, after it is pre-processed to be compatible with the model. Then, fine-tuning training can be started with the same code as in StyleTTS 2.

Training Hardware

For the results found in this post, VM with an Nvidia L4 GPU was used for training. Inference can however be done on a much less powerful machine. With our training dataset (~4500 sound files) it took about ~100 hours of training to train for 30 epochs with a batch size of 2. The “max length” is a configurable hyperparameter that can be reduced to increase batch size and reduce training time, but it was left at the default 400 for this reproduction.

Fine-tuning vs Training from scratch

At first, training the model from scratch was attempted, but it was quickly determined that this would take many more resources (compute and time) than we had available. Instead, we decided to fine tune an English LJSpeech-based model pretrained by the authors of the original paper. We then did single-speaker fine-tuning on this model using our dataset. This quickly provided reasonable results, well before the intended end of training at 30 epochs.

Training Summary

Given the (limited) expertise of the authors, the language chosen was French, but the following procedure should be the same for any language so long as a pre-trained model for that language is supplied. The steps to be followed are:

  1. Switch the PLBERT folder for the folder of the new pre-trained module. Alternatively, one can simply add the additional folder and re-route all of the references to PLBERT to the new folder path.
  2. Download training data for the model. This data should include .wav files with the audio and the corresponding .txt files with the transcript. In our case, we selected the SIWIS French Speech Synthesis Database.
  3. Create a .txt file with the phonemes corresponding to the text in the .txt files. These new files should be formatted in the following manner:

[name_of_the_wav_file]|[phonemes]|[speaker_number]

For our implementation, the phonemes were obtained by using the application “espeak” through the command terminal. However, the python library “phonemizer” may also be used to interface with espeak.

  1. Shuffle the entries in the previous file and assign training and testing split. For our code, we made this split 99% - 1% as it was close to the original distribution from the paper.
  2. Configure, or create the config file for the new dataset, PL-BERT, and capabilities of the training machine.
  3. Run the fine-tuning training using either (a) for multiple GPUs, or (b) if using a single GPU for accelerated training
  1. python train_finetune.py --config_path ./Configs/config_ft.yml
  2. accelerate launch --mixed_precision=fp16 --num_processes=1 train_finetune_accelerate.py --config_path ./Configs/config_ft.yml

Inference

Inference can be done similarly to StyleTTS 2. The only difference is that the phonemizer must be set to French, the multilingual PL-Bert must be loaded, and the newly trained French model must be used.

Conclusion

Our objective was to prove whether or not StyleTTS2 was capable of supporting multilingual text-to-speech conversion. The sample size that was interviewed was much smaller than that of the original paper, however, the findings from the survey prove that the model was satisfactorily trained in the French language. It is thus our conclusion that, with the correct pre-trained modules, StyleTTS2 can be used for multilingual text-to-speech.
